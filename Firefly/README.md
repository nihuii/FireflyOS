# Sleep Icon Change Log

## 本次功能

为 FireflyOS 的息屏界面增加了 4 张循环显示的流萤图片，素材来源于 `image/sleep/` 目录下的 4 张 PNG。

## 当前行为逻辑

1. 首次进入息屏时显示第 1 张图。
2. 只有在“锁屏界面进入息屏界面”时，才会切换到下一张图。
3. 4 张图片按顺序循环显示，切到最后一张后会回到第 1 张。
4. 如果屏幕已经进入黑屏状态，此时按 `BOOT` 键唤醒回到息屏界面，只会亮屏并恢复当前图片，不会切换到下一张。

## 本次界面调整

- 在息屏界面新增了图片控件，用于显示息屏图标。
- 为避免和图标重叠，息屏时间与日期位置做了下移调整。

## 原因分析

上一版实现为了追求更低运行时开销，使用了自定义的离线图片处理链路：

- 先把 PNG 缩放到 200x200。
- 再手动转换成 `RGB565 + Alpha` 的 LVGL 内置图片格式。

这条链路虽然运行时更省，但颜色处理不是走 LVGL 官方 PNG 解码流程，会受到自定义缩放、颜色量化和 Alpha 处理的影响，所以最终屏幕上有一部分颜色和原图不一致。

## 当前实现方式

现在改为：

- 先把 4 张图片预缩放为 200x200 的 PNG。
- 直接把 PNG 原始字节嵌入 `SleepIcons.cpp`。
- 显示时交给 LVGL 自带的 PNG 解码器处理颜色转换。

这样做的好处：

- 颜色显示和 PNG 本身保持一致，不再依赖手写像素转换逻辑。
- 仍然保留 200x200 资源尺寸，避免直接解码 510x510 原图带来更高内存与耗时。
- 图片只在切换时才需要重新解码，日常息屏显示仍然比较轻量。
- 相比上一版，资源体积也从约 2.96MB 降到了约 1.45MB。

## 涉及文件

- `Firefly/SleepIcons.h`
  - 息屏图片资源声明与获取接口。

- `Firefly/SleepIcons.cpp`
  - 4 张息屏图标的 PNG 原始字节资源。

- `Firefly/FireflyApp.h`
  - 息屏图片控件声明，并引入息屏图片头文件。

- `Firefly/FireflyState.cpp`
  - 息屏图片控件全局对象定义。

- `Firefly/FireflyInteraction.cpp`
  - 息屏图片索引状态。
  - 息屏图片刷新逻辑。
  - “锁屏进入息屏时切下一张图”的逻辑放在 `enter_sleep_screen_mode()` 中。
  - `wake_sleep_screen_from_blackout()` 只负责黑屏唤醒，不切图。

- `Firefly/Firefly.ino`
  - 在息屏页面中加入图片控件。
  - 调整息屏时间和日期文字位置。

## 备注

- 当前切图不是定时动画，而是按状态切换，因此系统负载更低。
- 当前方案在保证颜色正确性的前提下，仍然尽量遵守“更省资源、更快响应、不明显影响流畅度”的开发原则。
