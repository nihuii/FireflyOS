# Sleep Icon Change Log

## 本次功能

为 FireflyOS 的息屏界面增加了 4 张循环显示的流萤图片，素材来源于 `image/sleep/` 目录下的 4 张 PNG。

同时为锁屏界面增加了壁纸背景，当前使用第 1 张锁屏壁纸图片作为背景，资源格式现已改为原生 `RGB565`。

桌面界面也增加了壁纸背景，当前使用壁纸目录中的第 2 张图片作为桌面壁纸，资源格式同样改为原生 `RGB565`。

## 当前行为逻辑

1. 首次进入息屏时显示第 1 张图。
2. 只有在“锁屏界面进入息屏界面”时，才会切换到下一张图。
3. 4 张图片按顺序循环显示，切到最后一张后会回到第 1 张。
4. 如果屏幕已经进入黑屏状态，此时按 `BOOT` 键唤醒回到息屏界面，只会亮屏并恢复当前图片，不会切换到下一张。
5. 锁屏界面可以向上滑入桌面，但桌面界面不能再滑回锁屏。

## 本次界面调整

- 在息屏界面新增了图片控件，用于显示息屏图标。
- 为避免和图标重叠，息屏时间与日期位置做了下移调整。
- 桌面界面改为“壁纸 + 小应用图标”风格，不再使用原来的大卡片入口。
- 桌面当前只显示一个 `Settings` 小应用图标入口，但仍保留应用网格式布局，方便后续继续扩展其他应用。

## 本次性能优化

为优化“锁屏滑入桌面”时的卡顿，本次又做了两项调整：

- 将锁屏壁纸和桌面壁纸从 PNG 原始字节资源改为原生 `RGB565` 静态资源。
- 将 LVGL 图片缓存 `LV_IMG_CACHE_DEF_SIZE` 从 `1` 调整为 `2`。
- 将桌面小应用图标改为“滑动过程中隐藏，进入桌面后再显示”。
- 去掉桌面全屏蒙版。
- 去掉桌面图标阴影。
- 将 LVGL draw buffer 目标高度从 `80` 行提升到 `120` 行，并保留 `80` 行回退兜底。
- 新增最小风险双核方案：UI/LVGL 固定单核，后台任务固定另一核，仅处理 `BOOT` 短按检测与息屏时序判定。

这样做的好处：

- 锁屏和桌面两张全屏壁纸在滑动过程中不再触发 PNG 解码。
- 两张壁纸可以同时留在图片缓存中，减少滑动时反复顶掉缓存、重复处理的情况。
- 锁屏滑入桌面时，不再同时绘制桌面图标层，减少滑动过程中的对象重绘压力。
- 桌面页少了一层全屏透明混合和多组阴影绘制，软件渲染压力更低。
- 提高 draw buffer 后，单次刷新的行数更多，滑动时分块刷屏次数更少。
- `BOOT` 短按检测与自动息屏/黑屏时序不再和 `lv_timer_handler()` 争用同一条主循环。
- 运行时占用更低，更符合“更省资源、更快响应、不明显影响流畅度”的开发原则。

双核方案当前刻意保持保守边界：

- 后台核不直接调用任何 `lv_obj_*` / `lv_label_set_text` / `lv_img_set_src` 等 LVGL 接口。
- 后台核也不直接访问 PMU / RTC 的 I2C 读取逻辑，避免和 UI 核上的状态刷新产生总线竞争。
- 后台核只负责置位事件，真正的界面动作仍由 UI 核执行。

Arduino IDE `Tools` 中和双核相关的两项，当前建议也补充说明如下：

- `Arduino Runs On` 保持当前默认即可。在这套工程里，`setup()` / `loop()` / `lv_timer_handler()` 所在的 UI 主链路跑在哪个核心，后台任务就会自动固定到另一个核心。
- 如果当前板卡包的默认值是 `Core 0`，那么实际效果就是：UI / LVGL 在 `Core 0`，后台任务在 `Core 1`。
- `Events Run On` 当前也保持默认即可。它控制的是 Arduino 事件任务，不是这里手动创建的后台 FreeRTOS 任务。
- 由于当前工程没有把 Arduino 事件回调作为主要性能瓶颈，而且为了降低跨核 UI 调用风险，`Events Run On` 不需要为了这次双核优化单独改动。

## 原因分析

上一版实现为了追求更低运行时开销，使用了自定义的离线图片处理链路：

- 先把 PNG 缩放到 200x200。
- 再手动转换成 `RGB565 + Alpha` 的 LVGL 内置图片格式。

这条链路虽然运行时更省，但颜色处理不是走 LVGL 官方 PNG 解码流程，会受到自定义缩放、颜色量化和 Alpha 处理的影响，所以最终屏幕上有一部分颜色和原图不一致。

## 当前实现方式

现在改为：

- 先把 4 张图片预缩放为 200x200 的 PNG。
- 直接把 PNG 原始字节嵌入 `SleepIcons.cpp`。
- 显示时交给 LVGL 自带的 PNG 解码器处理颜色转换。
- 锁屏壁纸与桌面壁纸则单独使用原生 `RGB565` 静态资源，不再走 PNG 解码链路。

这样做的好处：

- 颜色显示和 PNG 本身保持一致，不再依赖手写像素转换逻辑。
- 仍然保留 200x200 资源尺寸，避免直接解码 510x510 原图带来更高内存与耗时。
- 图片只在切换时才需要重新解码，日常息屏显示仍然比较轻量。
- 相比上一版，资源体积也从约 2.96MB 降到了约 1.45MB。

## 涉及文件

- `Firefly/LockWallpaper.h`
  - 锁屏壁纸资源声明。

- `Firefly/LockWallpaper.cpp`
  - 锁屏壁纸与桌面壁纸原生 `RGB565` 资源。

- `Firefly/SleepIcons.h`
  - 息屏图片资源声明与获取接口。

- `Firefly/SleepIcons.cpp`
  - 4 张息屏图标的 PNG 原始字节资源。

- `Firefly/FireflyApp.h`
  - 息屏图片控件声明，并引入息屏图片头文件。

- `Firefly/FireflyState.cpp`
  - 息屏图片控件全局对象定义。

- `Firefly/FireflyInteraction.cpp`
  - 息屏图片索引状态。
  - 息屏图片刷新逻辑。
  - “锁屏进入息屏时切下一张图”的逻辑放在 `enter_sleep_screen_mode()` 中。
  - `wake_sleep_screen_from_blackout()` 只负责黑屏唤醒，不切图。
  - `tileview` 滑动开始时隐藏桌面图标层，滑动结束并真正进入桌面后再显示。
  - 新增固定到另一核心的后台任务。
  - 后台任务只负责 `BOOT` 短按事件与自动息屏/黑屏时序检测。
  - UI 核通过事件标志消费后台任务结果，保持 LVGL 单核访问。

- `Firefly/Firefly.ino`
  - 在息屏页面中加入图片控件。
  - 调整息屏时间和日期文字位置。
  - 在锁屏页面中加入壁纸图片控件。
  - 桌面页面改为壁纸加图标布局。
  - 禁止从桌面反向滑回锁屏。
  - 桌面图标改为独立图层，便于在滑动过程中整体隐藏。
  - 去掉桌面全屏蒙版与图标阴影。
  - 将 draw buffer 目标从 `80` 行提升到 `120` 行，并在内存不足时回退到 `80` 行。
  - 在 `setup()` 中启动后台核心任务。

- `libraries/lv_conf.h`
  - 将 LVGL 图片缓存数量从 `1` 调整为 `2`。

## 备注

- 当前切图不是定时动画，而是按状态切换，因此系统负载更低。
- 当前方案在保证颜色正确性的前提下，仍然尽量遵守“更省资源、更快响应、不明显影响流畅度”的开发原则。
